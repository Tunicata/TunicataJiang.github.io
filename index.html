Create New Plugin

So far, we have talked about two code files that may be useful in your Python automation process - Python files and C++ files. Fortunately, Plugin also has a Blueprint Function Library template, which is perfect for us to warp up our works.

A new Plugin would contain three files, the .uplugin file, Content folder, Resource folder, and Source folder. Content Folder will be the place to store the Python scripts, and the Source Folder will keep the C++ Source codes. Except for the Public and Private folders, there also have a ***.Build.cs** file under Source. Note that this is a C# file that supports the Plugin’s compilation, and before importing any UE libraries, you need first to claim the dependencies in this file.

Startup Scripts

A way to run a Python script, which I didn’t mention before, is the init_unreal.py File. If the Editor detects a script file called init_unreal.py in any of the paths it is configured to use, it automatically runs that script immediately. And The Content/Python sub-folders under any enabled Plugins’ folders are a part of them. In other words, you could write an init_unreal.py file, including your Plugin’s initialization steps.

Python Path Management in UE4

Posted on 2022-04-08 Edited on 2022-04-20 In Unreal Python

If you have worked with Python before, you should already know a famous Python command import. The Python documentation explains the import with two operations: 1. searches for the named model. 2. binds the search results to a name in the local scope. And the search path for modules is specified by a system-specific parameter sys.path. We went through how to run a simple script in the Unreal Editor, but the import between different scripts and modules is inevitable in our future works. Thus, how to manage them would be the topic for this post.

Python Paths Adds Through the Unreal Editor

Add Automatically

The Unreal Editor will automatically adds some paths to the sys.path. The way to check them is simple. We could directly read the sys.path parameter in the project’s Python Console.

The official document explains those automatically added paths into four different types:

  • The Content/Python sub-folder under your Project’s folder.
  • The Content/Python sub-folder in the main Unreal Engine installation.
  • The Content/Python sub-folder under each enabled Plugin’s folder.
  • The Documents/UnrealEngine/Python folder inside your user directory. For example, on Windows 10, this is equivalent to C:/Users/Username/Documents/UnrealEngine/Python

Before we want to add any new paths in sys.path, we should consider these folders first.

Add Your Own

In the last posts, we also introduced the Python setting under Edit > Project Settings >Plugins > Python. Despite the Startup Scripts setting, you could also find the Additional Paths setting. The paths added to it would also be automatically added to sys.path every time you open the project. But like the Startup Scripts, it is also hard to be transferred between multiple projects.

Other Ways to Add Python Paths

Environment Variable

PYTHONPATH is an environment variable that augments the module file’s default search path. The sys.path parameter would be initialized from the environment variable PYTHONPATH. Add paths to this value would also add them to the Python paths in the Unreal Editor.

Python Command

We could also add paths directly to the sys.path list with codes in Python script or commands in the Python console.

More on Paths

The import system of Python is powerful but easy to trigger unexpected bugs. When importing a module, the first module with the entered name would be imported. Therefore, sometimes you may import a wrong module to your script. But every Python module also has a variable file which could be used to check the path of the module you imported. Furthermore, the file variable could also be used in your scripts to add paths in a relatively dynamic way. Introduction to Python in UE 4

Posted on 2022-03-28 Edited on 2022-04-20 In Unreal Python

Python has been a widely used “glue” programming language in the media and entertainment industry. Most creative 3D production software provides Python API to help artists and developers optimize their production pipelines. Unreal Engine, a game engine developed by Epic Games, also allows developers to employ Python to improve their workflow. In this post, I will go through some background knowledge and setups necessary before you try to write and run your first Python script in Unreal Engine 4.

Background

What kind of problems could be solved by Python in UE4?

For the tasks you could accomplish with Python, I’d like to quote some words from the UE4’s official document:

  • Construct larger-scale asset management pipelines or workflows that tie the Unreal Editor to other 3D applications that you use in your organization.
  • Automate time-consuming Asset management tasks in the Unreal Editor, like generating Levels of Detail (LODs) for Static Meshes.
  • Procedurally lay out content in a Level.
  • Control the Unreal Editor from UIs that you create yourself in Python.

In short, Python is nearly a panacea except for writing the game logic itself. Python could successfully access most C++ APIs provided by UE4, which means it has almost the same power C++ has. At the same time, as a famous glue language, Python also has a large number of packages that could help developers complete some seemingly complicated works. For example, employing AI in animation automation with TensorFlow, managing your assets with highly visualized files like Xmind, or achieving tasks across different software.

Get Start

Enable plugin

Since Python is still an experimental feature in UE4, the Python Plugin in UE4 is not enabled by default. You can find the Plugins window in any UE4 project, and the Python Editor Script Plugin we need is under the Scripting section.

  • Edit > Plugins > Built-in > Scripting

Every Plugin under this section offers some APIs for the Editor, keeping all of them enabled as long as you may need them. Click their Enabled box and Restart the Editor to enable the Plugins. The version of the embedded Python in the Python Plugin is 3.7.7, and you can find it under your [UE4 main folder]\Engine\Binaries\ThirdParty\Python3\Win64.

You will also need the Output Log for future debugging. You can enable it in your UE4 Editor by choosing Window > Developer Tools > Output Log.

Set-up Your IDE

The last thing before we start is about the IDE, which is the Editor you will use in your future works. I recommend Microsoft Visual Studio Code since it is free and powerful enough to satisfy most scenarios. Other well-known Python IDE like PyCharm is also suitable for this task. For how to enable the autocomplete for UE4 Python scripting, you can find a detailed tutorial on UE4 documentation.
